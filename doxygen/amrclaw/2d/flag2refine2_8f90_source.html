1 ! ::::::::::::::::::::: flag2refine ::::::::::::::::::::::::::::::::::
2 !
3 ! User routine to control flagging of points for refinement.
4 !
5 ! Default version computes spatial difference dq in each direction and
6 ! for each component of q and flags any point where this is greater than
7 ! the tolerance tolsp. This is consistent with what the routine errsp did in
8 ! earlier versions of amrclaw (4.2 and before).
9 !
10 ! This routine can be copied to an application directory and modified to
11 ! implement some other desired refinement criterion.
12 !
13 ! Points may also be flagged for refining based on a Richardson estimate
14 ! of the error, obtained by comparing solutions on the current grid and a
15 ! coarsened grid. Points are flagged if the estimated error is larger than
16 ! the parameter tol in amr2ez.data, provided flag_richardson is .true.,
17 ! otherwise the coarsening and Richardson estimation is not performed!
18 ! Points are flagged via Richardson in a separate routine.
19 !
20 ! Once points are flagged via this routine and/or Richardson, the subroutine
21 ! flagregions is applied to check each point against the min_level and
22 ! max_level of refinement specified in any "region" set by the user.
23 ! So flags set here might be over-ruled by region constraints.
24 !
25 ! q = grid values including ghost cells (bndry vals at specified
26 ! time have already been set, so can use ghost cell values too)
27 !
28 ! aux = aux array on this grid patch
29 !
30 ! amrflags = array to be flagged with either the value
31 ! DONTFLAG (no refinement needed) or
32 ! DOFLAG (refinement desired)
33 !
34 ! tolsp = tolerance specified by user in input file amr2ez.data, used in default
35 ! version of this routine as a tolerance for spatial differences.
36 !
37 ! ::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
